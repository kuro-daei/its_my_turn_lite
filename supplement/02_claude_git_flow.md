# 補足 02. Claude と Git の連携フロー（詳細）

> **位置づけ:** この補足資料は [02_claude_usage.md](../02_claude_usage.md) の発展内容です。
> Git の基本操作（[06_git_internals.md](../06_git_internals.md)）と

## 概要

Claude Code は Git コマンドを代わりに実行できます。
「何をコミットすべきか」「コミットメッセージは何か」なども相談できます。

---

## 典型的な連携フロー（ASCII 図）

```
┌──────────────────────────────────────────────────────────────┐
│                  Claude Code × Git 連携フロー                  │
└──────────────────────────────────────────────────────────────┘

  ① 作業開始
  ─────────────────────────────────────────────────────────
  ユーザー: 「新しいブランチを切って作業を始めたい」
       │
       ▼
  Claude: git switch -c feature/xxx を実行
          ブランチ作成を確認して報告

  ② ファイル編集
  ─────────────────────────────────────────────────────────
  ユーザー: 「〇〇のファイルを作って・編集して」
       │
       ▼
  Claude: ファイルを作成・編集
          完了後に変更内容を報告

  ③ 変更内容の確認
  ─────────────────────────────────────────────────────────
  ユーザー: 「変更内容を確認して」
       │
       ▼
  Claude: git diff を実行
          差分を解説しながら表示

  ④ コミット
  ─────────────────────────────────────────────────────────
  ユーザー: 「今回の変更をコミットして」
       │
       ▼
  Claude: git add → git commit を実行
          コミットメッセージを自動生成
          （CLAUDE.md のルールに従う）

  ⑤ プッシュ・PR 作成
  ─────────────────────────────────────────────────────────
  ユーザー: 「GitHub にプッシュして PR を作って」
       │
       ▼
  Claude: git push → gh pr create を実行
          PR のタイトル・本文も自動生成

  ⑥ レビュー対応
  ─────────────────────────────────────────────────────────
  ユーザー: 「PR のコメントに対応して」
       │
       ▼
  Claude: コメント内容を確認して修正
          追加コミット → プッシュ
```

---

## Claude への Git 指示の例

```
# 現在の状態を確認
> git の状態を確認して変更ファイルを教えて

# コミット
> 今の変更内容を見て、適切なコミットメッセージを提案してコミットして

# ブランチ操作
> feature/user-auth というブランチを作って切り替えて

# PR 作成
> この変更内容で GitHub に PR を作って。
  タイトルと説明文は日本語で、変更の目的がわかるように書いて

# ログ確認
> 直近 5 件のコミット履歴を見やすく表示して

# コンフリクト解消
> main とのコンフリクトが出た。内容を確認して解消方法を提案して
```

---

## Git 操作を Claude に任せるときの注意点

| 注意事項 | 理由 |
|---|---|
| main への直接プッシュは指示しない | 本番コードに影響する可能性がある |
| `force push` は慎重に | 他の人のコミットが消える恐れがある |
| コミット前に差分を確認する | 意図しないファイルが含まれることがある |
| PR 本文は自分で読んでから作成する | 誤った情報が書かれることがある |

> **コツ:** CLAUDE.md にコミットメッセージのルール（Conventional Commits など）を
> 書いておくと、Claude が自動的にそのルールに従ってコミットメッセージを生成します。
> また、「main への直接プッシュは禁止」と書いておくと、Claude も遠慮するようになります。
