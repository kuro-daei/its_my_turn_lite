## このカリキュラムについて

このカリキュラムは、プログラムを書いたことがない人が「本番品質の Web アプリを自分で作る」ことを目標に設計されています。対象者はエンジニアではなく、プロダクトマネージャー、デザイナー、ビジネス職など、ソフトウェアの「使う側」にいた人たちです。

カリキュラムを通じて作るのは、実際に運用できるレベルの Web アプリケーションです。「動くかどうかわからないもの」ではなく、認証（ログイン機能）・データベース・デプロイ（インターネット上への公開）まで含んだ、本番に耐えるソフトウェアを構築します。その主役となるのが **Claude Code** です。

Claude Code を使えば、コマンドや構文を丸暗記しなくても、自分の言葉で指示を出しながら開発を進められます。ただし、魔法のように何でも解決してくれるわけではありません。「何を作りたいか」「なぜそうしたいか」を言語化する力と、開発の基本的な流れを理解しておくことが、このカリキュラムを使いこなす鍵です。

## 生成 AI の次のステップ

### 処理の中に AI を組み込む

AI 活用の第一世代は、既存のシステムやサービスの処理フローに AI を組み込むアプローチでした。画像の中に何が映っているかを自動で判別したり、大量のテキストから感情や意図を読み取ったり、ユーザーの行動履歴からおすすめを提示したりといった使い方です。このアプローチでは、AI は「既存の処理をよりスマートにするパーツ」として機能してきました。

こうした活用は引き続き重要ですが、このカリキュラムで扱うのは、もう一歩進んだ使い方です。

### AI を使ってソリッドなものを作る

Claude Code に代表される新しいアプローチでは、AI を「開発者の代行」として活用します。AI が何かを生成して、それ自体がプロダクトになるのではありません。AI が書いたコードをもとに、**決定論的で堅牢なソフトウェアそのものを構築する**スタイルです。

「決定論的」とは、「同じ操作をすれば必ず同じ結果になる」という意味です。Web アプリのボタンを押したら、毎回同じ処理が走る。これはソフトウェアの基本中の基本であり、AI が感覚で出力するものとは根本的に異なります。

このアプローチで生み出されるのはプロトタイプではなく、本番運用に耐えるソフトウェアです。学習コストは決して低くありません。「環境構築」「バージョン管理」「認証」といった概念の壁が最初にあります。しかし**この壁を越えれば、一人でも本番品質のソフトウェアを構築できる**ようになります。

### 誰がこのスキルを習得すべきか

既存のエンジニアがこのアプローチを習得するだけでは、本質的な価値は生まれにくい側面があります。エンジニアはすでに自分でコードを書けるからです。

真の価値が生まれるのは、**ビジネスの課題を肌で理解している人間が、自ら解決手段を構築できる**ようになったときです。「この業務フローを効率化したい」という課題感を持つ人が、その解決策を自分で設計・実装・運用できる。課題の言語化から設計・実装・運用までを一気通貫で担える人材が、最も大きなインパクトを生み出します。

このカリキュラムは、そういう人を増やすために作られています。
