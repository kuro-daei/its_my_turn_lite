## 1. チームでリサーチする流れ

```
[チーム全員で確認]
 役割分担を決める（例: Aさん → 企業概要、Bさん → 担当者情報）

          │
          ▼

[各自が自分の作業]
 担当ファイルを Claude でリサーチ → apple/info/ に保存

          │
          ▼

[全員で確認]
 お互いのファイルを git pull で取得して内容を確認
```

ポイントは**ファイルを分担すること**です。
同じファイルを同時に編集するとコンフリクト（衝突）が起きます（12章で体験します）。
まずは担当を決めてから作業しましょう。

---

## 2. 役割分担を決める

チームで次のように分担します。

| 担当者 | 作成するファイル | 内容 |
|---|---|---|
| Aさん（例: オーナー） | `apple/info/overview.md` | Apple 社の企業概要 |
| Bさん（例: メンバー） | `apple/info/products.md` | 主要製品・サービス一覧 |

3人チームの場合は、さらに追加分担できます。

| 担当者 | 作成するファイル | 内容 |
|---|---|---|
| Cさん | `apple/info/contacts.md` | 担当者情報テンプレート |

## 3. 各自が担当するファイルを Claude で作成

### Aさんの作業：企業概要（overview.md）

`~/works/apple` で Claude を起動した状態で、次のように依頼します。

```
> Apple 社について企業概要を調べて、以下の形式でまとめてほしい。

  ## Apple Inc. 企業概要

  **調査日**: 2026-03-09

  ### 基本情報
  - 正式名称:
  - 設立年:
  - 本社所在地:
  - 従業員数:
  - 売上高（最新）:
  - CEO:

  ### 事業内容
  （主要事業を3〜5文で）

  ### 主要製品・サービス
  （箇条書き5〜7点）

  ### 競合他社
  （主要競合を3社）
```

調査結果が出たら保存します。

```
> この内容を info/overview.md として保存して
```

```
info/overview.md を作成します。よいですか？ [y/n]
```

`y` を押すと保存されます。

```
【この時点の状態（Aさん）】
ワーキングツリー              ローカルRepo       GitHub (main)
  info/overview.md ★                             （まだ空）
  （保存済み・未コミット）
```

### Bさんの作業：製品情報（products.md）

`~/works/apple` で Claude を起動した状態で、次のように依頼します。

```
> Apple 社の主要製品・サービスをまとめて、以下の形式で整理してほしい。

  ## Apple 主要製品・サービス一覧

  **調査日**: 2026-03-09

  ### ハードウェア製品
  （製品名・概要・最新モデルを表形式で）

  ### ソフトウェア・サービス
  （主要サービス名・概要を箇条書きで）

  ### 売上構成（最新）
  （製品カテゴリ別の売上割合）
```

調査結果が出たら保存します。

```
> この内容を info/products.md として保存して
```

```
【この時点の状態（Bさん）】
ワーキングツリー              ローカルRepo       GitHub (main)
  info/products.md ★                             （まだ空）
  （保存済み・未コミット）
```

## 4. 互いの成果を確認する

> **注意:** この章では main ブランチに直接 push します。
> 08章以降でブランチを使った安全な方法を学びます。Branch Protection Rules（10章）を設定すると直接 push はできなくなります。

各自がファイルを保存したら、チームメンバーの変更を取得して確認し合います。

### 最新の変更を取得する

まず各自がコミット・push します（Claude への指示）。

```
> info/ の変更をコミットして、push して。
  メッセージは「feat: Apple の企業概要を追加」で（Aさんの場合）
```

```
> info/ の変更をコミットして、push して。
  メッセージは「feat: 製品情報を追加」で（Bさんの場合）
```

```
【全員が push 後の状態】

Aさんのローカル (main)    Bさんのローカル (main)
  overview.md               products.md
       │ push                    │ push
       ▼                         ▼
            GitHub (main)
              overview.md  ← Aさんが push
              products.md  ← Bさんが push
```

push が完了したら、お互いの変更を pull します。

```
> 最新の変更を pull して
```

```
✓ リモートから変更を取得しました
  info/overview.md  （Aさんが追加）
  info/products.md  （Bさんが追加）
```

```
【全員が pull 後の状態】
Aさんのローカル (main)     GitHub (main)      Bさんのローカル (main)
  overview.md              overview.md          overview.md ★（新規取得）
  products.md ★（新規取得） products.md          products.md
  ◀── pull ────────────────────────────────── pull ──▶
```

### 相手のファイルを確認する

```
> info/overview.md の内容を見せて
```

```
> info/products.md の内容を見せて
```

内容に気になる点があれば、その場で Claude に修正を依頼できます。

```
> info/overview.md の「事業内容」の部分をもう少し詳しくして
```

## 5. まとめ

この章で作成した成果物：

```
apple/
└── info/
    ├── overview.md    ← Aさんが作成（Apple 企業概要）
    └── products.md    ← Bさんが作成（主要製品・サービス一覧）
```

| ステップ | Claude への指示例 |
|---|---|
| リサーチ | `> Apple 社の企業概要を調べて、Markdown でまとめて` |
| ファイル保存 | `> info/overview.md として保存して` |
| コミット・push | `> info/ の変更をコミットして push して` |
| 最新取得 | `> 最新の変更を pull して` |
| 内容確認 | `> info/products.md の内容を見せて` |

チームで分担することで、短時間で多くの情報を整理できます。
次章では、この成果物を GitHub に公開する方法を学びます。
