## 1. Claude のリサーチ活用

### 単純検索との違い

Google 検索でできることと、Claude に依頼するリサーチの違いを理解することが第一歩です。

| 観点 | Google 検索 | Claude リサーチ |
|---|---|---|
| 操作 | キーワードを入力してリンクを開く | 自然言語で質問・依頼する |
| 情報の取得形式 | リンク一覧（自分で読む） | 要約・整理済みの文章 |
| 複数ソースの統合 | 手動で読み比べる必要がある | 自動的に統合・比較 |
| 追加質問 | 再検索が必要 | 同じ会話内で深掘り可能 |
| 出力のカスタマイズ | 不可 | Markdown・表・箇条書き等を指定可能 |
| 専門用語の解説 | 別途調べる必要がある | 調査と同時に解説してもらえる |

### Claude リサーチが特に有効なシーン

- 「〇〇について概要を把握したい」という初期調査
- 複数の概念を比較・対比したいとき
- 調査結果を即座にレポートや Markdown にまとめたいとき
- 担当者情報・部署構成を整理したいとき

## 2. Web 検索との連携

### MCP と WebSearch ツールとは

Claude は標準の言語モデルとしての知識だけでなく、**ツール（Tool）** を通じて外部情報にアクセスできます。

- **MCP（Model Context Protocol）**: Claude がファイルシステム、API、データベースなど外部リソースと連携するための標準プロトコル
- **WebSearch ツール**: MCP を通じて Claude がリアルタイムの Web 検索を実行できる機能

```
[あなた]
   ↓ 質問・依頼
[Claude]
   ↓ 必要と判断したらツールを呼び出す
[WebSearch / MCP Server]
   ↓ 検索結果・データを返す
[Claude]
   ↓ 結果を解釈・統合して回答
[あなた]
```

### Claude Code での Web 検索の使い方

Claude に話しかけるだけで自動的に Web 検索が走ります。

```
> 博報堂の最近のニュースをまとめて
```

Claude が必要と判断したタイミングで自動的に Web 検索を実行し、結果を統合して回答します。
「Web 検索して」と明示しなくても、最新情報が必要な質問には自動で検索が走ります。

### プロンプト設計のコツ

リサーチの精度は、プロンプトの書き方で大きく変わります。

**悪い例: 漠然としすぎている**

```
> 博報堂について教えて
```

問題点:
- 何を知りたいのかが不明
- 範囲が広すぎて表面的な回答になる

**良い例: 目的・範囲・形式を明確にする**

```
> 博報堂について以下の項目をまとめて。
  Markdown の箇条書きで出力してほしい。
  - 事業内容（50字以内）
  - 主要サービス・ブランド（3〜5個）
  - 本社所在地・設立年
  - 従業員数（グループ全体）
  - 主要グループ会社
```

## 3. 企業調査：博報堂の概要をまとめる

Claude に次のように依頼してみましょう。

```
> 博報堂について企業概要を調べて、以下の形式でまとめてほしい。

  ## 博報堂 企業概要

  **調査日**: 2026-03-09

  ### 基本情報
  - 正式名称:
  - 設立年:
  - 本社所在地:
  - 従業員数（グループ全体）:
  - 売上高（最新）:

  ### 事業内容
  （広告・マーケティング事業の概要を3〜5文で）

  ### 主要サービス・強み
  （箇条書き3〜5点）

  ### 主要グループ会社
  （箇条書き3〜5社）

  ### 競合他社
  （国内主要競合を2〜3社）
```

Claude が Web 検索を使いながら、構造化されたレポートを生成します。

## 4. 担当者情報をまとめる

取引先の担当者情報も Claude で整理できます。
実際の名前・連絡先は社内情報として持っているはずなので、
ここでは **フォーマット（テンプレート）** を Claude に作ってもらいましょう。

```
> 博報堂の担当者情報を管理するための Markdown テンプレートを作って。
  以下の情報を含む表形式で。

  - 部署名
  - 担当者名
  - 役職
  - メールアドレス
  - 電話番号
  - 備考（性格・好みなど）
```

Claude が次のようなテンプレートを作成します。

```
## 博報堂 担当者一覧

**最終更新**: 2026-03-09

| 部署 | 担当者名 | 役職 | メール | 電話 | 備考 |
|---|---|---|---|---|---|
| 営業部 | 山田 太郎 | 部長 | yamada@hakuhodo.co.jp | 03-XXXX-XXXX | 打ち合わせは午前中希望 |
| クリエイティブ局 | 鈴木 花子 | ディレクター | suzuki@hakuhodo.co.jp | 03-XXXX-XXXX | Slack 連絡が早い |
```

実際の担当者情報を追記して、自社内の情報として活用してください。

## 5. 調査結果を hakuhodo/info/ に保存する

調査結果を `info/` フォルダに保存します。Claude への指示だけで完結します。

### 企業概要を保存する

```
> さっきまとめた博報堂の企業概要を info/overview.md として保存して
```

Claude がファイルを作成します。

```
info/overview.md を作成します。よいですか？ [y/n]
```

`y` を押すとファイルが保存されます。

### 担当者一覧を保存する

```
> 担当者テンプレートを info/contacts.md として保存して
```

保存完了後、フォルダ構成を確認してみましょう。

```
> info/ フォルダの中身を見せて
```

```
info/
├── overview.md    ← 企業概要
└── contacts.md    ← 担当者一覧
```

### VS Code でプレビュー確認

VS Code でファイルを開き、`Ctrl + Shift + V`（Mac: `Cmd + Shift + V`）を押すと
Markdown がプレビュー表示されます。Markdown の詳しい使い方は [12_markdown.md](./12_markdown.md) で学びます。

## 6. まとめ

### リサーチ → Markdown 化のフロー

```
1. 調査テーマを決める
   └─ 「博報堂の企業概要を把握したい」

         │
         ▼

2. プロンプトを設計する
   ├─ 目的・範囲・出力形式を指定
   └─ 必要な項目をリストアップ

         │
         ▼

3. Claude に調査依頼
   ├─ [知識のみ] 学習データ範囲内の調査
   └─ [Web 検索有効] 最新情報も含む調査

         │
         ▼

4. 回答を評価・深掘り
   ├─ 不明点を追加質問
   └─ 必要なら別角度から再質問

         │
         ▼

5. ファイルに保存
   ├─ 「info/overview.md として保存して」と Claude に依頼
   └─ VS Code でプレビューして見た目を確認
```

### リサーチ精度を上げる 3 つのポイント

1. **具体的に聞く**: 「博報堂について教えて」より「博報堂の主要事業を表形式でまとめて」
2. **出力形式を指定する**: Markdown・表・箇条書きなど形式を明示する
3. **段階的に深掘りする**: 1 回の質問に詰め込まず、会話を重ねる
