## 1. はじめに：ターミナルとは

### GUI と CLI の違い

パソコンを操作する方法には、大きく分けて 2 種類あります。

| 比較項目 | GUI（グラフィカルUI） | CLI（コマンドラインインタフェース） |
|---|---|---|
| 操作方法 | マウスでアイコンをクリック | キーボードで命令文を入力 |
| 代表例 | Windows エクスプローラー、Finder | ターミナル、コマンドプロンプト |
| 学習コスト | 直感的で低い | コマンドを覚える必要がある |
| 作業速度 | 慣れるとそこそこ | 慣れると非常に速い |
| 自動化 | 難しい | スクリプトで簡単に自動化できる |
| 再現性 | 手順を文章で伝えにくい | コマンドをそのままコピーして再現できる |

### なぜ CLI を学ぶのか

開発現場では CLI が標準です。理由はいくつかあります。

- **再現性**: コマンドをそのままメモや README に書けば、他の人も同じ操作ができる
- **速度**: 慣れれば GUI よりはるかに速く操作できる
- **自動化**: シェルスクリプトで繰り返し作業を自動化できる
- **サーバー操作**: クラウドサーバーなどは GUI がなく、CLI のみが提供される

### ターミナルを開く（WSL の場合）

Windows + WSL 環境では、以下のいずれかの方法でターミナルを開きます。

- スタートメニューから「Ubuntu」を検索して起動
- Windows Terminal をインストールして Ubuntu タブを開く
- VS Code を開き、メニューの「ターミナル」→「新しいターミナル」を選ぶ

起動すると、以下のようなプロンプトが表示されます。

```
eiji@DESKTOP-ABC123:~$
```

各部分の意味は次のとおりです。

| 部分 | 意味 |
|---|---|
| `eiji` | ログイン中のユーザー名 |
| `DESKTOP-ABC123` | コンピューター名（ホスト名） |
| `~` | 現在いる場所（カレントディレクトリ） |
| `$` | コマンド入力待ち（`#` は root ユーザーを示す） |

`$` の右にカーソルが点滅していれば、コマンドを入力できる状態です。

## 2. カレントディレクトリという概念

### ファイルシステムの木構造

コンピューターのファイルは、木（ツリー）のような階層構造で管理されています。
一番上の `/`（ルートディレクトリ）を起点に、枝分かれしていくイメージです。

本章を進めると、最終的に次の構造が出来上がります。

```
/
└── home/
    └── eiji/                    ← ホームディレクトリ（~ と表記）
        └── works/               ← 作業フォルダ（この章で作る）
            └── hakuhodo/        ← 博報堂プロジェクト（この章で作る）
                ├── README.md    ← プロジェクトの説明ファイル
                └── info/        ← 調査メモを置くフォルダ
```

Windows に慣れている方は `C:\Users\eiji\works` のような書き方を見たことがあると思います。
Linux/WSL では `/home/eiji/works` のように **スラッシュ `/`** を使って階層を表します。

> [!NOTE]
> **WSL（Ubuntu）のホームディレクトリは最初ほぼ空です**
>
> WSL を起動したばかりのホームディレクトリには、`.bashrc` や `.profile` などの
> 隠しファイル（ドットファイル）しかありません。
>
> **macOS の場合:** `Desktop/`, `Documents/`, `Downloads/` などが最初から存在します。

### カレントディレクトリ = 「今いる場所」

ターミナルを操作するとき、あなたは常にファイルシステムのどこか一点にいます。
これを **カレントディレクトリ**（current directory、現在のディレクトリ）と呼びます。

Windows エクスプローラーで、今開いているフォルダがカレントディレクトリに相当します。

### pwd でカレントディレクトリを確認する

`pwd` は **P**rint **W**orking **D**irectory の略です。
「今どこにいるか」を表示する最もシンプルなコマンドです。

```bash
pwd
```

```
/home/eiji
```

ターミナルを開いた直後は、ホームディレクトリ（`/home/eiji`）にいます。
迷子になったら、まず `pwd` を実行して現在地を確認しましょう。

### 絶対パス・相対パス・チルダ（~）の違い

ファイルやディレクトリの「場所」を指定する方法は 3 種類あります。

#### 絶対パス

`/` から始まる、一意に定まる場所の指定方法です。
どこにいても同じ場所を指します。

```
/home/eiji/works/hakuhodo/README.md
```

#### 相対パス

「今いる場所」を起点にした場所の指定方法です。

```bash
# カレントディレクトリが /home/eiji のとき
works/hakuhodo/README.md    # → /home/eiji/works/hakuhodo/README.md を指す
```

相対パスの特殊な記法：

| 記法 | 意味 |
|---|---|
| `.` | カレントディレクトリ自身 |
| `..` | 一つ上（親）のディレクトリ |
| `../..` | 二つ上のディレクトリ |

#### チルダ（~）

`~`（チルダ）は、ログイン中のユーザーのホームディレクトリを表す短縮記法です。

| 書き方 | 展開先の例 |
|---|---|
| `~` | `/home/eiji` |
| `~/works` | `/home/eiji/works` |
| `~/works/hakuhodo` | `/home/eiji/works/hakuhodo` |

## 3. cd でディレクトリを移動する

### 基本操作

`cd` は **C**hange **D**irectory の略です。
ターミナル上での「移動」に使います。

```bash
cd works
```

実行後は `pwd` で移動できたか確認する習慣をつけましょう。

```bash
# 移動前の確認
pwd
```

```
/home/eiji
```

```bash
# 移動する
cd works
```

```bash
# 移動後の確認
pwd
```

```
/home/eiji/works
```

### cd のパターン表

よく使う `cd` のパターンをまとめました。

| コマンド | 意味 | 使い場面 |
|---|---|---|
| `cd ~` | ホームディレクトリへ移動 | どこにいてもホームに戻りたいとき |
| `cd` | ホームディレクトリへ移動（`cd ~` と同じ） | 素早くホームに戻るとき |
| `cd ..` | 一つ上のディレクトリへ移動 | 親ディレクトリに戻りたいとき |
| `cd ../..` | 二つ上のディレクトリへ移動 | 複数階層さかのぼりたいとき |
| `cd -` | 直前にいたディレクトリへ移動 | 2 か所を行き来したいとき |
| `cd /home/eiji/works` | 絶対パスで移動 | どこからでも確実に移動したいとき |
| `cd hakuhodo` | 相対パスで移動 | 子ディレクトリに移動するとき |

`cd ..` の動きを図で確認しましょう。

```
移動前：/home/eiji/works/hakuhodo にいる

/home/eiji/
└── works/
    └── hakuhodo/    ← ここにいる
        ├── README.md
        └── info/

cd .. を実行

移動後：/home/eiji/works にいる

/home/eiji/
└── works/           ← ここに移動した
    └── hakuhodo/
```

### よくあるミスと対処法

#### ミス 1：スペルミス

```bash
cd hakuhodo    # スペルミスがあると
```

```
bash: cd: hakuhodo: No such file or directory
```

対処法：`Tab` キーで補完しましょう（詳しくは [6 章](#6-ターミナル操作のコツ)）。

#### ミス 2：大文字・小文字の違い

Linux は **大文字・小文字を区別します**。

```bash
cd Hakuhodo    # 間違い（大文字）
```

```
bash: cd: Hakuhodo: No such file or directory
```

```bash
cd hakuhodo    # 正しい（小文字）
```

対処法：`ls` でディレクトリ名を確認してから入力するか、`Tab` 補完を使いましょう。

#### ミス 3：現在地がわからなくなった

どこにいるかわからなくなったら、まず `pwd` で現在地を確認し、必要であれば `cd ~` でホームに戻りましょう。

```bash
pwd             # 現在地を確認
cd ~            # ホームに戻る
```

## 4. ls でファイル一覧を確認する

### 基本の ls

`ls` は **L**i**s**t の略です。カレントディレクトリの中身を一覧表示します。

ホームディレクトリで実行すると：

```bash
ls
```

```
works
```

`works` の中を見るには：

```bash
ls works
```

```
hakuhodo
```

### オプションで表示内容を変える

#### ls -l（詳細表示）

```bash
ls -l ~/works/hakuhodo
```

```
total 4
-rw-r--r-- 1 eiji eiji  38 Mar  8 10:00 README.md
drwxr-xr-x 2 eiji eiji 512 Mar  8 10:00 info
```

各列の意味：

| 列 | 意味 | 例 |
|---|---|---|
| 1列目 | ファイル種別とパーミッション | `drwxr-xr-x` |
| 3列目 | 所有者（ユーザー） | `eiji` |
| 5列目 | ファイルサイズ（バイト） | `38` |
| 6〜8列目 | 最終更新日時 | `Mar  8 10:00` |
| 9列目 | ファイル名 | `README.md` |

1 列目の読み方：

```
drwxr-xr-x
│└─────────── パーミッション（アクセス権）
└──────────── ファイル種別（d=ディレクトリ、-=ファイル）
```

#### ls -a（隠しファイルも表示）

ファイル名が `.` で始まるファイルは通常非表示です（隠しファイル）。
`-a` オプションで表示できます。

```bash
ls -a ~
```

```
.  ..  .bash_logout  .bashrc  .profile  works
```

#### ls -la（詳細 + 隠しファイル）

最もよく使う組み合わせです。

```bash
ls -la ~/works/hakuhodo
```

```
total 8
drwxr-xr-x 3 eiji eiji  512 Mar  8 10:00 .
drwxr-xr-x 3 eiji eiji  512 Mar  8 10:00 ..
-rw-r--r-- 1 eiji eiji   38 Mar  8 10:00 README.md
drwxr-xr-x 2 eiji eiji  512 Mar  8 10:00 info
```

よく使うオプション一覧：

| オプション | 意味 |
|---|---|
| `ls` | シンプルな一覧 |
| `ls -l` | 詳細表示（パーミッション・サイズ・日時） |
| `ls -a` | 隠しファイルも表示 |
| `ls -la` | 詳細 + 隠しファイル |
| `ls -lh` | 詳細 + ファイルサイズを人間が読みやすい形式で（KB, MB） |

## 5. ファイルとディレクトリの基本操作

ここからは、実際に手を動かしてプロジェクトフォルダを作っていきます。
**以下のコマンドを上から順に実行してください。**

### Step 1：作業フォルダを作る

まずホームディレクトリにいることを確認します。

```bash
pwd
```

```
/home/eiji
```

`works` フォルダを作ります。`mkdir` は **M**a**k**e **Dir**ectory の略です。

```bash
mkdir ~/works
ls
```

```
works
```

> [!WARNING]
> `mkdir` を実行しても何も表示されなければ、成功しています。
> エラーが表示されたときだけ失敗です。`ls` で確認しましょう。

### Step 2：プロジェクトフォルダを作る

`works` に移動して、博報堂プロジェクト用のフォルダを作ります。

```bash
cd ~/works
mkdir hakuhodo
ls
```

```
hakuhodo
```

`hakuhodo` の中に入り、情報を置く `info` フォルダも作ります。

```bash
cd hakuhodo
mkdir info
ls
```

```
info
```

現在地を確認しましょう。

```bash
pwd
```

```
/home/eiji/works/hakuhodo
```

### Step 3：README ファイルを作って書き込む

`touch` で空のファイルを作成します。

```bash
touch README.md
ls -l
```

```
total 0
-rw-r--r-- 1 eiji eiji 0 Mar  8 10:00 README.md
drwxr-xr-x 2 eiji eiji 512 Mar  8 10:00 info
```

サイズが `0` になっているのは、中身のない空ファイルだからです。

`echo` でテキストを書き込みます。`>` を使うとファイルに**上書き**保存できます。

```bash
echo "# 博報堂 プロジェクト" > README.md
cat README.md
```

```
# 博報堂 プロジェクト
```

`>>` を使うと、既存の内容を消さずに**追記**できます。

```bash
echo "博報堂との取引情報・調査メモをここに集める" >> README.md
cat README.md
```

```
# 博報堂 プロジェクト
博報堂との取引情報・調査メモをここに集める
```

| 記号 | 動作 | 元の内容 |
|---|---|---|
| `>` | 上書き | 消える |
| `>>` | 追記 | 保持される |

> [!WARNING]
> `>` を間違えて使うと、大切なファイルの内容が消えます。
> 重要なファイルに書き込む前に、ファイル名をよく確認しましょう。

### Step 4：間違って作ったファイルを削除する

作業中に不要なファイルを作ってしまうことがあります。

```bash
touch temp.txt
ls
```

```
README.md  info  temp.txt
```

`rm`（**R**e**m**ove）で削除します。

```bash
rm temp.txt
ls
```

```
README.md  info
```

ディレクトリを削除するには `-r` オプションが必要です。

```bash
mkdir old-info
rm -r old-info
ls
```

```
README.md  info
```

> [!WARNING]
> `rm` で削除したファイルは **ゴミ箱に入らず、即座に完全削除** されます。
> Windows のゴミ箱のように「元に戻す」ことはできません。
>
> 特に `rm -rf /` や `rm -rf *` は絶対に実行しないでください。
>
> 削除前には必ず `ls` でファイル名を確認し、慎重に実行してください。

`-i` オプションを使うと、削除前に確認を求めてくれます。

```bash
rm -i README.md
```

```
rm: remove regular file 'README.md'?
```

`n` + Enter でキャンセルしましょう（README.md は残しておきます）。

### Step 5：完成したフォルダ構造を確認する

```bash
ls -la ~/works/hakuhodo
```

```
total 8
drwxr-xr-x 3 eiji eiji  512 Mar  8 10:00 .
drwxr-xr-x 3 eiji eiji  512 Mar  8 10:00 ..
-rw-r--r-- 1 eiji eiji   38 Mar  8 10:00 README.md
drwxr-xr-x 2 eiji eiji  512 Mar  8 10:00 info
```

プロジェクトの土台ができました。

```
~/works/hakuhodo/
├── README.md    ← プロジェクトの説明
└── info/        ← 調査メモを置く場所
```

## 6. ターミナル操作のコツ

コマンドを効率よく入力するためのテクニックを紹介します。

### Tab 補完

コマンドやファイル名の途中まで入力して `Tab` キーを押すと、自動で補完されます。

```
cd ha[Tab]
→ cd hakuhodo/
```

候補が複数ある場合は、もう一度 `Tab` を押すと候補一覧が表示されます。

この 2 つの使い方を覚えましょう。

1. **スペルミスの防止**: Tab 補完を使えばタイプミスがなくなる
2. **存在確認**: Tab で補完できるなら、そのファイルは存在する

### 矢印キーで履歴を呼び出す

- `↑` キーで前に入力したコマンドを呼び出せます
- `↓` キーで新しいほうに戻ります

### Ctrl ショートカット表

| ショートカット | 動作 | よく使う場面 |
|---|---|---|
| `Ctrl + C` | 実行中のコマンドを強制終了 | フリーズしたときや、長い処理を止めたいとき |
| `Ctrl + L` | 画面をクリア | ターミナルが出力でいっぱいになったとき |
| `Ctrl + A` | 行頭にカーソルを移動 | 長いコマンドの先頭を修正したいとき |
| `Ctrl + E` | 行末にカーソルを移動 | 長いコマンドの末尾に戻りたいとき |
| `Ctrl + W` | カーソル前の単語を削除 | コマンドの一部を素早く消したいとき |
| `Ctrl + U` | カーソルより前を全部削除 | 入力をやり直したいとき |
| `Ctrl + R` | コマンド履歴を検索 | 昔使ったコマンドを探したいとき |
| `Ctrl + D` | ターミナルのセッションを終了 | ターミナルを閉じたいとき |

### コマンドとオプションのヘルプを見る

コマンドの使い方がわからなくなったら `--help` を試しましょう。

```bash
ls --help
```

詳細なマニュアルは `man` コマンドで見られます（`q` キーで終了）。

```bash
man ls
```

## 7. まとめ：学んだコマンド一覧

本章で学んだコマンドをまとめます。

| コマンド | 書き方の例 | 意味 |
|---|---|---|
| `pwd` | `pwd` | カレントディレクトリを表示する |
| `cd` | `cd hakuhodo` | ディレクトリを移動する |
| `cd ~` | `cd ~` | ホームディレクトリへ移動 |
| `cd ..` | `cd ..` | 一つ上のディレクトリへ移動 |
| `cd -` | `cd -` | 直前のディレクトリへ移動 |
| `ls` | `ls` | ファイル一覧を表示する |
| `ls -l` | `ls -l` | 詳細情報付きで表示する |
| `ls -la` | `ls -la` | 隠しファイルも含めて詳細表示 |
| `mkdir` | `mkdir info` | ディレクトリを作成する |
| `touch` | `touch README.md` | 空のファイルを作成する |
| `cat` | `cat README.md` | ファイルの内容を表示する |
| `echo` | `echo "text"` | 文字列を出力する |
| `echo >` | `echo "text" > README.md` | ファイルに上書き書き込み |
| `echo >>` | `echo "text" >> README.md` | ファイルに追記 |
| `rm` | `rm temp.txt` | ファイルを削除する（元に戻せない） |
| `rm -r` | `rm -r old-info` | ディレクトリを削除する |
| `rm -i` | `rm -i README.md` | 削除前に確認を求める |

### キーボードショートカット早見表

| 操作 | ショートカット |
|---|---|
| コマンドを強制終了 | `Ctrl + C` |
| 画面をクリア | `Ctrl + L` |
| 行頭に移動 | `Ctrl + A` |
| 行末に移動 | `Ctrl + E` |
| 前の単語を削除 | `Ctrl + W` |
| 前の入力を全削除 | `Ctrl + U` |
| 履歴を検索 | `Ctrl + R` |
| Tab 補完 | `Tab` |
| 前のコマンドを呼び出す | `↑` |
