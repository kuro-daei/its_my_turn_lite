## 1. Markdown とは

**Markdown（マークダウン）** は、テキストファイルに軽量な記号を加えるだけで、見出しや箇条書きなどの構造を表現できる記法です。

### なぜ調査ノートに使うのか

| 特徴 | メリット |
|------|----------|
| プレーンテキスト | Git で差分管理しやすい |
| 学習コストが低い | 記号数が少なく覚えやすい |
| 可搬性が高い | エディタを選ばず開ける |
| GitHub がレンダリング | push するとそのまま HTML として表示される |

### GitHub での表示

`.md` 拡張子のファイルを GitHub にプッシュすると、リポジトリページで自動的に HTML に変換されて表示されます。`README.md` はリポジトリのトップページに表示されるため、プロジェクトの顔になります。

VS Code では `Ctrl + Shift + V` でプレビューを開くことができます。

## 2. 基本的な記法

### 見出し

`#` の数でレベルを指定します。`#` の後に半角スペースを入れるのがルールです。

**Markdown ソース:**

```markdown
# 見出し 1（H1）
## 見出し 2（H2）
### 見出し 3（H3）
#### 見出し 4（H4）
```

**表示結果のイメージ:**

- `# 見出し 1` → 最も大きい見出し（ページタイトル相当）
- `## 見出し 2` → セクション見出し
- `### 見出し 3` → サブセクション見出し
- `#### 見出し 4` → さらに細かい分類

> **コツ:** 文書全体で `#`（H1）は 1 つだけにするのが慣習です。H2 以下でセクションを整理しましょう。

---

### 段落

空行を挟むと段落が分かれます。単なる改行（空行なし）は段落を分けません。

**Markdown ソース:**

```markdown
これは最初の段落です。
この行は同じ段落に続きます。

ここから新しい段落が始まります。
```

**表示結果:**

これは最初の段落です。この行は同じ段落に続きます。

ここから新しい段落が始まります。

---

### 強調（太字・斜体）

**Markdown ソース:**

```markdown
**太字**にするには asterisk を 2 つ重ねます。
*斜体*にするには asterisk を 1 つ使います。
***太字かつ斜体***にするには 3 つ使います。
~~取り消し線~~はチルダ 2 つです。
```

**表示結果:**

- `**太字**` → **太字**
- `*斜体*` → *斜体*
- `***太字かつ斜体***` → ***太字かつ斜体***
- `~~取り消し線~~` → ~~取り消し線~~

> **実用例:** エラーメッセージや重要な用語を **太字** にすると読みやすくなります。

---

### インラインコード

文章の中にコマンドや変数名を書くときは、バッククォート（`` ` ``）で囲みます。

**Markdown ソース:**

```markdown
`git status` を実行して状態を確認します。
変数 `userName` に名前を代入してください。
```

**表示結果:**

`git status` を実行して状態を確認します。変数 `userName` に名前を代入してください。

## 3. リスト

### 箇条書きリスト（順序なし）

行頭に `- ` または `* ` または `+ ` を付けます。

**Markdown ソース:**

```markdown
- りんご
- バナナ
- みかん
```

**表示結果:**

- りんご
- バナナ
- みかん

---

### 番号付きリスト（順序あり）

行頭に `1. ` のような数字とピリオドを付けます。番号は実際の値より表示上の順序が優先されるため、すべて `1.` と書いても正しく連番になります。

**Markdown ソース:**

```markdown
1. リポジトリを作成する
2. ローカルにクローンする
3. ファイルを編集する
4. コミットしてプッシュする
```

**表示結果:**

1. リポジトリを作成する
2. ローカルにクローンする
3. ファイルを編集する
4. コミットしてプッシュする

---

### ネスト（入れ子）リスト

子リストは **スペース 2 つ** または **スペース 4 つ** でインデントします。

**Markdown ソース:**

```markdown
- Git の基本操作
  - commit（変更を記録する）
  - push（リモートに送る）
  - pull（リモートから取得する）
- GitHub の機能
  - Pull Request（レビュー依頼）
  - Issues（課題管理）
```

**表示結果:**

- Git の基本操作
  - commit（変更を記録する）
  - push（リモートに送る）
  - pull（リモートから取得する）
- GitHub の機能
  - Pull Request（レビュー依頼）
  - Issues（課題管理）

---

### タスクリスト（チェックボックス）

GitHub では `- [ ]` でチェックボックス、`- [x]` でチェック済みを表示できます。

**Markdown ソース:**

```markdown
- [x] WSL をインストールした
- [x] Git の初期設定をした
- [ ] SSH キーを GitHub に登録した
- [ ] 最初のリポジトリを作成した
```

**表示結果（GitHub 上）:**

- [x] WSL をインストールした
- [x] Git の初期設定をした
- [ ] SSH キーを GitHub に登録した
- [ ] 最初のリポジトリを作成した

> **活用例:** 学習の進捗チェックや、作業の TODO リストに使うと便利です。

## 4. コードブロック

複数行のコードを表示するには、バッククォート 3 つ（` ``` `）で囲みます。

### 基本構文

**Markdown ソース:**

````markdown
```
echo "Hello, World!"
```
````

**表示結果:**

```
echo "Hello, World!"
```

---

### シンタックスハイライト

バッククォートの後に言語名を指定すると、構文に合わせた色付き表示になります。

**Markdown ソース（bash の例）:**

````markdown
```bash
# ブランチを作成して切り替え\ngit switch -c feature/add-notes

# ステージングしてコミット
git add my_notes.md
git commit -m "feat: 調査メモを追加"

# リモートにプッシュ
git push -u origin feature/add-notes
```
````

**表示結果:**

```bash
# ブランチを作成して切り替え
git switch -c feature/add-notes

# ステージングしてコミット
git add my_notes.md
git commit -m "feat: 調査メモを追加"

# リモートにプッシュ
git push -u origin feature/add-notes
```

---

### よく使う言語指定

| 指定キーワード | 言語・用途 |
|----------------|------------|
| `bash` / `sh`  | シェルスクリプト・コマンド |
| `python`       | Python コード |
| `javascript`   | JavaScript |
| `json`         | JSON データ |
| `yaml`         | YAML 設定ファイル |
| `markdown`     | Markdown ソースを見せるとき |
| `diff`         | 差分表示 |
| （なし）       | 汎用・言語不明 |

> **コツ:** ターミナルで実行するコマンドには必ず `bash` を指定しましょう。読者がコマンドとコードを見分けやすくなります。

## 5. リンク・画像

### リンク

**Markdown ソース:**

```markdown
[表示テキスト](URL)

[GitHub](https://github.com)
[コマンド早見表を見る](./supplement/git/04_command_reference.md)
```

**表示結果:**

- `[GitHub](https://github.com)` → [GitHub](https://github.com)
- `[コマンド早見表を見る](./supplement/git/04_command_reference.md)` → 同じリポジトリ内のファイルへのリンクとして機能

---

### ページ内リンク（アンカーリンク）

見出しに自動的にアンカーが生成されます。`#` の後に見出しテキストを小文字・ハイフン区切りで書きます。

**Markdown ソース:**

```markdown
[目次に戻る](#目次)
[コードブロックのセクションへ](#4-コードブロック)
```

> **注意:** 日本語の見出しのアンカーはそのまま使えます。スペースはハイフン `-` に置き換わります。

---

### 画像

リンクと同じ構文に `!` を先頭に付けます。

**Markdown ソース:**

```markdown
![代替テキスト](画像のURL または相対パス)

![Git のワークフロー図](./images/git-workflow.png)
![GitHub のロゴ](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)
```

**書き方のポイント:**

- `[]` の中の **代替テキスト** は、画像が読めない場合に表示されます。スクリーンリーダーのためにも必ず書きましょう。
- リポジトリ内の画像は相対パスで参照できます。

---

### URL をそのまま表示する

**Markdown ソース:**

```markdown
<https://github.com>
```

`<` と `>` で囲むと、クリック可能なリンクとしてそのままの URL が表示されます。

## 6. テーブル

### 基本構文

列をパイプ `|` で区切り、2 行目にハイフン `-` で区切り線を書きます。

**Markdown ソース:**

```markdown
| コマンド | 説明 |
|----------|------|
| `git status` | 作業ツリーの状態を確認 |
| `git add` | 変更をステージング |
| `git commit` | コミットを作成 |
| `git push` | リモートに送信 |
```

**表示結果:**

| コマンド | 説明 |
|----------|------|
| `git status` | 作業ツリーの状態を確認 |
| `git add` | 変更をステージング |
| `git commit` | コミットを作成 |
| `git push` | リモートに送信 |

---

### 列の配置（左寄せ・中央・右寄せ）

区切り線の書き方で配置を指定できます。

**Markdown ソース:**

```markdown
| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| Apple  | Orange   | Banana |
| 100    | 200      | 300    |
```

**表示結果:**

| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| Apple  | Orange   | Banana |
| 100    | 200      | 300    |

| 区切り記法 | 配置 |
|------------|------|
| `|------|` | 左寄せ（デフォルト） |
| `|:-----|` | 左寄せ（明示） |
| `|:----:|` | 中央寄せ |
| `|-----:|` | 右寄せ |

---

### 整形のコツ

テーブルはソース上でパイプを揃えなくても動作しますが、揃えると読みやすくなります。VS Code では **Markdown Table Formatter** などの拡張機能を使うと自動整形できます。

**整形前:**

```markdown
| 名前 | 年齢 | 職業 |
|---|---|---|
| 田中 | 25 | エンジニア |
| 鈴木 | 30 | デザイナー |
```

**整形後（見やすい）:**

```markdown
| 名前 | 年齢 | 職業       |
|------|------|------------|
| 田中 | 25   | エンジニア |
| 鈴木 | 30   | デザイナー |
```

## 7. 引用

### 基本的な引用（blockquote）

行頭に `> ` を付けると引用ブロックになります。

**Markdown ソース:**

```markdown
> これは引用ブロックです。
> 複数行にわたって書くことができます。
```

**表示結果:**

> これは引用ブロックです。
> 複数行にわたって書くことができます。

---

### ネストした引用

`>` を重ねると入れ子の引用になります。

**Markdown ソース:**

```markdown
> 外側の引用です。
>
> > 内側の引用（返信のイメージ）です。
```

**表示結果:**

> 外側の引用です。
>
> > 内側の引用（返信のイメージ）です。

---

### blockquote の使いどころ

引用ブロックは「引用」だけでなく、補足・注意・ヒントを目立たせるためにも広く使われます。

| 使いどころ | 例 |
|------------|----|
| 公式ドキュメントからの引用 | `> Git is a free and open source...` |
| 注意書き | `> **注意:** この操作は元に戻せません。` |
| ヒント・補足 | `> **コツ:** ブランチ名は作業内容を表す名前にする。` |
| エラーメッセージの引用 | `> fatal: not a git repository` |

**実用例:**

```markdown
> **注意:** `git reset --hard` は作業ツリーの変更を完全に破棄します。
> コミット済みの変更以外は復元できないため、実行前に確認してください。
```

表示結果:

> **注意:** `git reset --hard` は作業ツリーの変更を完全に破棄します。
> コミット済みの変更以外は復元できないため、実行前に確認してください。

## 8. 実践：調査メモをブランチ経由で push する

ここまで学んだ Markdown 記法を使って、実際に調査メモを書いてブランチ経由で push する流れを練習します。

### ステップ 1: ブランチを作成する

```bash
# main ブランチを最新にしてから作業ブランチを作る
git pull
git switch -c feature/add-markdown-notes
```

### ステップ 2: Markdown ファイルを作成する

VS Code で新しいファイル `markdown_notes.md` を作成し、以下のような内容を書いてみましょう。

```markdown
# Markdown 調査メモ

## 今日学んだこと

- 見出しは `#` の数でレベルを指定する
- コードブロックには言語名を付けるとハイライトされる
- テーブルはパイプ `|` で列を区切る

## 疑問点

> Q: GitHub での Markdown と通常の Markdown に違いはある？
> A: GitHub Flavored Markdown（GFM）という拡張仕様があり、
>    タスクリストや絵文字などが追加されている。

## 次にやること

1. README.md を Markdown で書き直す
2. テーブルを使ってコマンド一覧を整理する
```

### ステップ 3: 変更をコミットする

```bash
# ファイルをステージング
git add markdown_notes.md

# コミット
git commit -m "feat: Markdown 調査メモを追加"
```

### ステップ 4: リモートに push する

```bash
git push -u origin feature/add-markdown-notes
```

### ステップ 5: GitHub で確認する

1. `https://github.com/[ユーザー名]/[リポジトリ名]` を開く
2. `feature/add-markdown-notes` ブランチに切り替える
3. `markdown_notes.md` をクリックして、Markdown が正しくレンダリングされているか確認する

### ステップ 6: Pull Request を作成する（オプション）

```bash
# GitHub CLI を使う場合
gh pr create --title "feat: Markdown 調査メモを追加" --body "学習メモを追加します。"
```

または GitHub の Web UI から「Compare & pull request」ボタンをクリックします。

## 9. まとめ（記法早見表）

### Markdown 記法早見表

| 記法 | Markdown ソース | 表示結果・説明 |
|------|----------------|----------------|
| **見出し H1** | `# 見出し` | 最大の見出し |
| **見出し H2** | `## 見出し` | セクション見出し |
| **見出し H3** | `### 見出し` | サブセクション |
| **太字** | `**テキスト**` | **テキスト** |
| **斜体** | `*テキスト*` | *テキスト* |
| **太字＋斜体** | `***テキスト***` | ***テキスト*** |
| **取り消し線** | `~~テキスト~~` | ~~テキスト~~ |
| **インラインコード** | `` `code` `` | `code` |
| **箇条書き** | `- 項目` | 順序なしリスト |
| **番号付きリスト** | `1. 項目` | 順序ありリスト |
| **ネストリスト** | `  - 子項目`（2 スペース） | 入れ子リスト |
| **タスクリスト** | `- [ ] 未完了` / `- [x] 完了` | チェックボックス |
| **コードブロック** | ` ```bash ` ... ` ``` ` | シンタックスハイライト付きコード |
| **リンク** | `[テキスト](URL)` | クリック可能なリンク |
| **画像** | `![代替テキスト](URL)` | 画像の埋め込み |
| **テーブル** | `\| 列1 \| 列2 \|` | 表の作成 |
| **左寄せ** | `\|:---\|` | 左揃え |
| **中央寄せ** | `\|:---:\|` | 中央揃え |
| **右寄せ** | `\|---:\|` | 右揃え |
| **引用** | `> テキスト` | 引用ブロック |
| **ネスト引用** | `>> テキスト` | 入れ子引用 |
| **水平線** | `---` | 区切り線 |
| **URL リンク** | `<https://...>` | URL をそのまま表示 |
| **ページ内リンク** | `[テキスト](#見出し)` | アンカーリンク |
| **段落** | 空行で区切る | 段落の区切り |

### VS Code でのプレビュー

| 操作 | ショートカット |
|------|----------------|
| Markdown プレビューを開く | `Ctrl + Shift + V` |
| エディタとプレビューを並べる | `Ctrl + K` → `V` |
| プレビューをリフレッシュ | ファイル保存（`Ctrl + S`）で自動更新 |

### GitHub Flavored Markdown（GFM）の特徴

GitHub では標準 Markdown を拡張した **GFM（GitHub Flavored Markdown）** が使われています。

| GFM の追加機能 | 説明 |
|----------------|------|
| タスクリスト | `- [ ]` / `- [x]` |
| テーブル | パイプ `\|` 記法 |
| 取り消し線 | `~~テキスト~~` |
| コードブロックのシンタックスハイライト | 言語名指定 |
| 絵文字 | `:thumbsup:` → 👍（GitHub 上でレンダリング） |
| @メンション | `@ユーザー名` → クリック可能なリンクになる（Issue/PR コメント内では通知も届く） |
| Issue 参照 | `#123` で同リポジトリの Issue や PR へのリンクになる |
