## 1. はじめに：ターミナルとは

### GUI と CLI の違い

パソコンを操作する方法には、大きく分けて 2 種類あります。

| 比較項目 | GUI（グラフィカルUI） | CLI（コマンドラインインタフェース） |
|---|---|---|
| 操作方法 | マウスでアイコンをクリック | キーボードで命令文を入力 |
| 代表例 | Windows エクスプローラー、Finder | ターミナル、コマンドプロンプト |
| 学習コスト | 直感的で低い | コマンドを覚える必要がある |
| 作業速度 | 慣れるとそこそこ | 慣れると非常に速い |
| 自動化 | 難しい | スクリプトで簡単に自動化できる |
| 再現性 | 手順を文章で伝えにくい | コマンドをそのままコピーして再現できる |

### なぜ CLI を学ぶのか

開発現場では CLI が標準です。理由はいくつかあります。

- **再現性**: コマンドをそのままメモや README に書けば、他の人も同じ操作ができる
- **速度**: 慣れれば GUI よりはるかに速く操作できる
- **自動化**: シェルスクリプトで繰り返し作業を自動化できる
- **サーバー操作**: クラウドサーバーなどは GUI がなく、CLI のみが提供される

### ターミナルを開く（WSL の場合）

Windows + WSL 環境では、以下のいずれかの方法でターミナルを開きます。

- スタートメニューから「Ubuntu」を検索して起動
- Windows Terminal をインストールして Ubuntu タブを開く
- VS Code を開き、メニューの「ターミナル」→「新しいターミナル」を選ぶ

起動すると、以下のようなプロンプトが表示されます。

```
eiji@DESKTOP-ABC123:~$
```

各部分の意味は次のとおりです。

| 部分 | 意味 |
|---|---|
| `eiji` | ログイン中のユーザー名 |
| `DESKTOP-ABC123` | コンピューター名（ホスト名） |
| `~` | 現在いる場所（カレントディレクトリ） |
| `$` | コマンド入力待ち（`#` は root ユーザーを示す） |

`$` の右にカーソルが点滅していれば、コマンドを入力できる状態です。

## 2. カレントディレクトリという概念

### ファイルシステムの木構造

コンピューターのファイルは、木（ツリー）のような階層構造で管理されています。
一番上の `/`（ルートディレクトリ）を起点に、枝分かれしていくイメージです。

```
/                          ← ルートディレクトリ（一番上）
├── home/                  ← ホームディレクトリの親
│   └── eiji/              ← ユーザー eiji のホームディレクトリ（最初はほぼ空）
│       ├── .bashrc        ← シェルの設定ファイル（隠しファイル）
│       ├── .profile       ← ログイン時の設定ファイル（隠しファイル）
│       └── works/      ← 自分で作る作業フォルダ（次のセクションで作成）
│           └── my-repo/   ← GitHub からクローンしたリポジトリ
├── etc/                   ← システム設定ファイル
├── usr/                   ← プログラムのインストール先
└── tmp/                   ← 一時ファイル
```

Windows に慣れている方は `C:\Users\eiji\works` のような書き方を見たことがあると思います。
Linux/WSL では `/home/eiji/works` のように **スラッシュ `/`** を使って階層を表します。

> **WSL（Ubuntu）のホームディレクトリは最初ほぼ空です**
>
> WSL を起動したばかりのホームディレクトリには、`.bashrc` や `.profile` などの
> 隠しファイル（ドットファイル）しかありません。
> Windows の `C:\Users\yourname\Documents` などのフォルダは
> `/mnt/c/Users/yourname/Documents` に別途マウントされています。
>
> **macOS の場合:** `Desktop/`, `Documents/`, `Downloads/` などが最初から存在します。

### カレントディレクトリ = 「今いる場所」

ターミナルを操作するとき、あなたは常にファイルシステムのどこか一点にいます。
これを **カレントディレクトリ**（current directory、現在のディレクトリ）と呼びます。

Windows エクスプローラーで、今開いているフォルダがカレントディレクトリに相当します。

```
エクスプローラーで「works」フォルダを開いている状態
      ↕（同じ概念）
ターミナルで /home/eiji/works にいる状態
```

### pwd でカレントディレクトリを確認する

`pwd` は **P**rint **W**orking **D**irectory の略です。
「今どこにいるか」を表示する最もシンプルなコマンドです。

```bash
pwd
```

実行結果の例：

```
/home/eiji
```

これは「今 `/home/eiji` というディレクトリにいる」という意味です。
迷子になったら、まず `pwd` を実行して現在地を確認しましょう。

### 最初にやること：作業フォルダを作る

ホームディレクトリは最初ほぼ空なので、まず作業用のフォルダを作りましょう。

```bash
mkdir ~/works
```

```bash
cd ~/works
pwd
```

```
/home/eiji/works
```

これで `~/works` を「プロジェクトの置き場」として使えます。
GitHub からリポジトリをクローンするときも、このフォルダの中で行います。

> **命名は自由:** `works` でも `works` でも `dev` でも構いません。
> 後の章（GitHub Publish）では、ここにリポジトリをクローンします。

### 絶対パス・相対パス・チルダ（~）の違い

ファイルやディレクトリの「場所」を指定する方法は 3 種類あります。

#### 絶対パス

`/` から始まる、一意に定まる場所の指定方法です。
どこにいても同じ場所を指します。

```
/home/eiji/works/my-repo/README.md
```

```
/（ルート）
└── home/
    └── eiji/
        └── works/
            └── my-repo/
                └── README.md   ← ここを指す
```

#### 相対パス

「今いる場所」を起点にした場所の指定方法です。
カレントディレクトリが変わると、同じ指定でも異なる場所を指すことがあります。

```
# カレントディレクトリが /home/eiji のとき
works/my-repo/README.md       # → /home/eiji/works/my-repo/README.md を指す
```

相対パスの特殊な記法：

| 記法 | 意味 |
|---|---|
| `.` | カレントディレクトリ自身 |
| `..` | 一つ上（親）のディレクトリ |
| `../..` | 二つ上のディレクトリ |

#### チルダ（~）

`~`（チルダ）は、ログイン中のユーザーのホームディレクトリを表す短縮記法です。

```
~ = /home/eiji（eiji としてログインしている場合）
```

| 書き方 | 展開先の例 |
|---|---|
| `~` | `/home/eiji` |
| `~/works` | `/home/eiji/works` |
| `~/works/my-repo` | `/home/eiji/works/my-repo` |

図で整理すると次のようになります。

```
/home/eiji/                  ← ~ （チルダ）はここ
│
├── works/                ← ~/works（絶対パスで /home/eiji/works）
│   └── my-repo/
│
└── works/
    └── project/             ← カレントディレクトリが ここ だとすると…
        └── src/             ← 相対パスで src/ 、絶対パスで /home/eiji/works/project/src/
```

## 3. cd でディレクトリを移動する

### 基本操作

`cd` は **C**hange **D**irectory の略です。
ターミナル上での「移動」に使います。

```bash
cd works
```

実行後は `pwd` で移動できたか確認する習慣をつけましょう。

```bash
# 移動前の確認
pwd
```

```
/home/eiji
```

```bash
# 移動する
cd works
```

```bash
# 移動後の確認
pwd
```

```
/home/eiji/works
```

### cd のパターン表

よく使う `cd` のパターンをまとめました。

| コマンド | 意味 | 使い場面 |
|---|---|---|
| `cd ~` | ホームディレクトリへ移動 | どこにいてもホームに戻りたいとき |
| `cd` | ホームディレクトリへ移動（`cd ~` と同じ） | 素早くホームに戻るとき |
| `cd ..` | 一つ上のディレクトリへ移動 | 親ディレクトリに戻りたいとき |
| `cd ../..` | 二つ上のディレクトリへ移動 | 複数階層さかのぼりたいとき |
| `cd -` | 直前にいたディレクトリへ移動 | 2 か所を行き来したいとき |
| `cd /home/eiji/works` | 絶対パスで移動 | どこからでも確実に移動したいとき |
| `cd works` | 相対パスで移動 | 子ディレクトリに移動するとき |

`cd ..` の動きを図で確認しましょう。

```
移動前：/home/eiji/works にいる

/home/
└── eiji/
    └── works/    ← ここにいる
        └── my-repo/

cd .. を実行

移動後：/home/eiji にいる

/home/
└── eiji/             ← ここに移動した
    └── works/
        └── my-repo/
```

`cd -` は便利な技です。

```bash
cd ~/works/my-repo    # プロジェクトへ移動
pwd
```

```
/home/eiji/works/my-repo
```

```bash
cd ~/works            # works へ移動
pwd
```

```
/home/eiji/works
```

```bash
cd -                     # 直前にいた場所へ戻る
pwd
```

```
/home/eiji/works/my-repo
```

### よくあるミスと対処法

#### ミス 1：スペルミス

```bash
cd projedts    # "works" をタイプミス
```

```
bash: cd: projedts: No such file or directory
```

対処法：`Tab` キーで補完しましょう（詳しくは [6 章](#6-ターミナル操作のコツ)）。

#### ミス 2：大文字・小文字の違い

Linux は **大文字・小文字を区別します**。

```bash
cd Projects    # 間違い（先頭が大文字）
```

```
bash: cd: Projects: No such file or directory
```

```bash
cd works    # 正しい（小文字）
```

対処法：`ls` でディレクトリ名を確認してから入力するか、`Tab` 補完を使いましょう。

#### ミス 3：ファイルへ cd しようとした

```bash
cd README.md    # ファイルは cd できない
```

```
bash: cd: README.md: Not a directory
```

対処法：`ls` でディレクトリかファイルかを確認しましょう。`cd` できるのはディレクトリだけです。

#### ミス 4：現在地がわからなくなった

どこにいるかわからなくなったら、まず `pwd` で現在地を確認し、必要であれば `cd ~` でホームに戻りましょう。

```bash
pwd             # 現在地を確認
cd ~            # ホームに戻る
```

## 4. ls でファイル一覧を確認する

### 基本の ls

`ls` は **L**i**s**t の略です。カレントディレクトリの中身を一覧表示します。

```bash
ls
```

```
（何も表示されない、またはディレクトリを作った後であれば works など）
```

`~/works` を作成した後にホームで実行した場合：

```bash
ls
```

```
works
```

特定のディレクトリの中身を見るには、パスを引数に指定します。

```bash
ls works
```

```
my-repo
```

### オプションで表示内容を変える

`ls` にはいくつかの便利なオプションがあります。

#### ls -l（詳細表示）

```bash
ls -l
```

```
total 4
drwxr-xr-x 2 eiji eiji 512 Mar  8 10:00 works
```

各列の意味：

| 列 | 意味 | 例 |
|---|---|---|
| 1列目 | ファイル種別とパーミッション | `drwxr-xr-x` |
| 2列目 | リンク数 | `2` |
| 3列目 | 所有者（ユーザー） | `eiji` |
| 4列目 | 所有グループ | `eiji` |
| 5列目 | ファイルサイズ（バイト） | `512` |
| 6〜8列目 | 最終更新日時 | `Mar  8 10:00` |
| 9列目 | ファイル名 | `works` |

1 列目の読み方：

```
drwxr-xr-x
│└─────────── パーミッション（アクセス権）
└──────────── ファイル種別（d=ディレクトリ、-=ファイル）
```

#### ls -a（隠しファイルも表示）

ファイル名が `.` で始まるファイルは通常非表示です（隠しファイル）。
`-a` オプションで表示できます。

```bash
ls -a
```

```
.  ..  .bash_logout  .bashrc  .profile  works
```

`.` がカレントディレクトリ、`..` が親ディレクトリを表します。

#### ls -la（詳細 + 隠しファイル）

最もよく使う組み合わせです。`-l` と `-a` を一緒に指定します。

```bash
ls -la
```

```
total 16
drwxr-xr-x 1 eiji eiji  512 Mar  8 10:00 .
drwxr-xr-x 1 root root 4096 Mar  8 09:00 ..
-rw-r--r-- 1 eiji eiji  220 Mar  8 09:00 .bash_logout
-rw-r--r-- 1 eiji eiji 3526 Mar  8 09:00 .bashrc
-rw-r--r-- 1 eiji eiji  807 Mar  8 09:00 .profile
drwxr-xr-x 2 eiji eiji  512 Mar  8 10:00 works
```

よく使うオプション一覧：

| オプション | 意味 |
|---|---|
| `ls` | シンプルな一覧 |
| `ls -l` | 詳細表示（パーミッション・サイズ・日時） |
| `ls -a` | 隠しファイルも表示 |
| `ls -la` | 詳細 + 隠しファイル |
| `ls -lh` | 詳細 + ファイルサイズを人間が読みやすい形式で（KB, MB） |

## 5. ファイルとディレクトリの基本操作

### mkdir でディレクトリを作る

`mkdir` は **M**a**k**e **Dir**ectory の略です。新しいディレクトリを作成します。

```bash
mkdir practice
```

```bash
ls
```

```
practice  works
```

一度に複数作ることもできます。

```bash
mkdir dir1 dir2 dir3
```

`-p` オプションを使うと、途中のディレクトリが存在しなくても一括で作れます。

```bash
mkdir -p works/project/src
```

これは以下と同じ操作をまとめて行います。

```
mkdir works
mkdir works/project
mkdir works/project/src
```

> **注意**: `mkdir` を実行しても何も表示されなければ、成功しています。
> エラーが表示されたときだけ失敗です。`ls` で確認しましょう。

### touch で空のファイルを作る

`touch` は、空のファイルを作成するコマンドです。

```bash
touch memo.txt
```

```bash
ls -l
```

```
-rw-r--r-- 1 eiji eiji 0 Mar  8 12:00 memo.txt
```

サイズが `0` になっているのは、中身のない空ファイルだからです。

複数ファイルを同時に作ることもできます。

```bash
touch file1.txt file2.txt file3.md
```

> **補足**: `touch` はもともと「ファイルの最終更新日時を現在時刻に更新する」コマンドです。
> ファイルが存在しない場合に限り、新しい空ファイルを作ります。

### cat でファイルの内容を表示する

`cat` は **con**caten**at**e（連結する）の略ですが、
主に「ファイルの内容を表示する」目的でよく使われます。

```bash
cat memo.txt
```

（空ファイルなので何も表示されない）

内容があるファイルの場合：

```bash
cat /etc/os-release
```

```
NAME="Ubuntu"
VERSION="22.04.3 LTS (Jammy Jellyfish)"
ID=ubuntu
...
```

### echo でテキストを出力・ファイルに書き込む

`echo` は文字列をターミナルに出力するコマンドです。

```bash
echo "Hello, World!"
```

```
Hello, World!
```

`>` と `>>` を使うと、出力先をファイルに変えることができます。

#### `>` で上書き書き込み

```bash
echo "はじめての行" > memo.txt
cat memo.txt
```

```
はじめての行
```

もう一度 `>` で書き込むと、**上書き**（元の内容が消える）されます。

```bash
echo "別の内容" > memo.txt
cat memo.txt
```

```
別の内容
```

元の「はじめての行」は消えました。

#### `>>` で追記

```bash
echo "1行目" > memo.txt
echo "2行目" >> memo.txt
echo "3行目" >> memo.txt
cat memo.txt
```

```
1行目
2行目
3行目
```

`>>` は既存の内容を保持したまま末尾に追記します。

| 記号 | 動作 | 元の内容 |
|---|---|---|
| `>` | 上書き | 消える |
| `>>` | 追記 | 保持される |

> **注意**: `>` を間違えて使うと、大切なファイルの内容が消えます。
> 重要なファイルに書き込む前に、ファイル名をよく確認しましょう。

### rm でファイルを削除する

`rm` は **R**e**m**ove の略です。ファイルを削除します。

```bash
rm memo.txt
```

```bash
ls
```

（memo.txt が消えている）

ディレクトリを削除するには `-r` オプションが必要です。

```bash
rm -r practice
```

> **警告**: `rm` で削除したファイルは **ゴミ箱に入らず、即座に完全削除** されます。
> Windows のゴミ箱のように「元に戻す」ことはできません。
>
> 特に `rm -rf /` や `rm -rf *` は絶対に実行しないでください。
> システム全体のファイルが削除され、回復不能になります。
>
> 削除前には必ず `ls` でファイル名を確認し、慎重に実行してください。

`-i` オプションを使うと、削除前に確認を求めてくれます。

```bash
rm -i memo.txt
```

```
rm: remove regular file 'memo.txt'?
```

`y` + Enter で削除、`n` + Enter でキャンセルです。

## 6. ターミナル操作のコツ

コマンドを効率よく入力するためのテクニックを紹介します。

### Tab 補完

コマンドやファイル名の途中まで入力して `Tab` キーを押すと、自動で補完されます。

```
cd pro[Tab]
→ cd works
```

候補が複数ある場合は、もう一度 `Tab` を押すと候補一覧が表示されます。

```
cd p[Tab][Tab]
→ practice/   works/
```

この 2 つの使い方を覚えましょう。

1. **スペルミスの防止**: Tab 補完を使えばタイプミスがなくなる
2. **存在確認**: Tab で補完できるなら、そのファイルは存在する

### 矢印キーで履歴を呼び出す

- `↑` キーで前に入力したコマンドを呼び出せます
- `↓` キーで新しいほうに戻ります

たとえば `pwd` を何度も実行したいとき、毎回タイプする必要はありません。

```
↑ キーを押す → pwd が表示される → Enter
```

### Ctrl ショートカット表

| ショートカット | 動作 | よく使う場面 |
|---|---|---|
| `Ctrl + C` | 実行中のコマンドを強制終了 | フリーズしたときや、長い処理を止めたいとき |
| `Ctrl + L` | 画面をクリア | ターミナルが出力でいっぱいになったとき |
| `Ctrl + A` | 行頭にカーソルを移動 | 長いコマンドの先頭を修正したいとき |
| `Ctrl + E` | 行末にカーソルを移動 | 長いコマンドの末尾に戻りたいとき |
| `Ctrl + W` | カーソル前の単語を削除 | コマンドの一部を素早く消したいとき |
| `Ctrl + U` | カーソルより前を全部削除 | 入力をやり直したいとき |
| `Ctrl + R` | コマンド履歴を検索 | 昔使ったコマンドを探したいとき |
| `Ctrl + D` | ターミナルのセッションを終了 | ターミナルを閉じたいとき |

### コマンド履歴を検索する（Ctrl + R）

`Ctrl + R` を押すと、過去のコマンド履歴をインクリメンタル検索できます。

```
(reverse-i-search)`git': git commit -m "feat: 追加"
```

キーワードを入力すると、マッチするコマンドが表示されます。
見つかったら `Enter` で実行、`Ctrl + C` でキャンセルします。

### その他の便利な使い方

#### コマンドとオプションのヘルプを見る

コマンドの使い方がわからなくなったら `--help` を試しましょう。

```bash
ls --help
```

詳細なマニュアルは `man` コマンドで見られます（`q` キーで終了）。

```bash
man ls
```

#### 複数コマンドの連続実行

`;` で区切ると、複数のコマンドを連続して実行できます。

```bash
mkdir newdir; cd newdir; pwd
```

```
/home/eiji/newdir
```
