## 1. Markdown とは

**Markdown（マークダウン）** は、テキストファイルに軽量な記号を加えるだけで、見出しや箇条書きなどの構造を表現できる記法です。

### なぜ調査ノートに使うのか

| 特徴 | メリット |
|------|----------|
| プレーンテキスト | Git で差分管理しやすい |
| 学習コストが低い | 記号数が少なく覚えやすい |
| 可搬性が高い | エディタを選ばず開ける |
| GitHub がレンダリング | push するとそのまま HTML として表示される |

### GitHub での表示

`.md` 拡張子のファイルを GitHub にプッシュすると、リポジトリページで自動的に HTML に変換されて表示されます。`README.md` はリポジトリのトップページに表示されるため、プロジェクトの顔になります。

VS Code では `Ctrl + Shift + V` でプレビューを開くことができます。

## 2. 基本的な記法

### 見出し

`#` の数でレベルを指定します。`#` の後に半角スペースを入れるのがルールです。

**Markdown ソース:**

```markdown
# 見出し 1（H1）
## 見出し 2（H2）
### 見出し 3（H3）
#### 見出し 4（H4）
```

**表示結果のイメージ:**

- `# 見出し 1` → 最も大きい見出し（ページタイトル相当）
- `## 見出し 2` → セクション見出し
- `### 見出し 3` → サブセクション見出し
- `#### 見出し 4` → さらに細かい分類

> **コツ:** 文書全体で `#`（H1）は 1 つだけにするのが慣習です。H2 以下でセクションを整理しましょう。

---

### 段落

空行を挟むと段落が分かれます。単なる改行（空行なし）は段落を分けません。

**Markdown ソース:**

```markdown
これは最初の段落です。
この行は同じ段落に続きます。

ここから新しい段落が始まります。
```

**表示結果:**

これは最初の段落です。この行は同じ段落に続きます。

ここから新しい段落が始まります。

---

### 強調（太字・斜体）

**Markdown ソース:**

```markdown
**太字**にするには asterisk を 2 つ重ねます。
*斜体*にするには asterisk を 1 つ使います。
***太字かつ斜体***にするには 3 つ使います。
~~取り消し線~~はチルダ 2 つです。
```

**表示結果:**

- `**太字**` → **太字**
- `*斜体*` → *斜体*
- `***太字かつ斜体***` → ***太字かつ斜体***
- `~~取り消し線~~` → ~~取り消し線~~

---

### インラインコード

文章の中にコマンドや変数名を書くときは、バッククォート（`` ` ``）で囲みます。

**Markdown ソース:**

```markdown
`git status` を実行して状態を確認します。
ファイル `info/overview.md` を確認してください。
```

**表示結果:**

`git status` を実行して状態を確認します。ファイル `info/overview.md` を確認してください。

## 3. リスト

### 箇条書きリスト（順序なし）

行頭に `- ` または `* ` または `+ ` を付けます。

**Markdown ソース:**

```markdown
- iPhone
- iPad
- Mac
```

**表示結果:**

- iPhone
- iPad
- Mac

---

### 番号付きリスト（順序あり）

行頭に `1. ` のような数字とピリオドを付けます。

**Markdown ソース:**

```markdown
1. リポジトリを作成する
2. ローカルにクローンする
3. ファイルを編集する
4. コミットしてプッシュする
```

**表示結果:**

1. リポジトリを作成する
2. ローカルにクローンする
3. ファイルを編集する
4. コミットしてプッシュする

---

### ネスト（入れ子）リスト

子リストは **スペース 2 つ** または **スペース 4 つ** でインデントします。

**Markdown ソース:**

```markdown
- Apple の主要事業
  - ハードウェア（iPhone・Mac・iPad）
  - ソフトウェア（iOS・macOS）
  - サービス（App Store・Apple Music）
```

**表示結果:**

- Apple の主要事業
  - ハードウェア（iPhone・Mac・iPad）
  - ソフトウェア（iOS・macOS）
  - サービス（App Store・Apple Music）

---

### タスクリスト（チェックボックス）

GitHub では `- [ ]` でチェックボックス、`- [x]` でチェック済みを表示できます。

**Markdown ソース:**

```markdown
- [x] info/overview.md を作成した
- [x] info/contacts.md を作成した
- [ ] meetings/2025-01-meeting.md を作成する
- [ ] PR をレビューする
```

**表示結果（GitHub 上）:**

- [x] info/overview.md を作成した
- [x] info/contacts.md を作成した
- [ ] meetings/2025-01-meeting.md を作成する
- [ ] PR をレビューする

## 4. コードブロック

複数行のコードを表示するには、バッククォート 3 つ（` ``` `）で囲みます。

### 基本構文

**Markdown ソース:**

````markdown
```
echo "Hello, World!"
```
````

**表示結果:**

```
echo "Hello, World!"
```

---

### シンタックスハイライト

バッククォートの後に言語名を指定すると、構文に合わせた色付き表示になります。

**Markdown ソース（bash の例）:**

````markdown
```bash
# 作業ディレクトリを確認して移動
TARGET_DIR=~/works/apple

if [ -d "$TARGET_DIR" ]; then
    echo "ディレクトリが見つかりました: $TARGET_DIR"
    cd "$TARGET_DIR"
else
    echo "エラー: $TARGET_DIR が存在しません"
    exit 1
fi
```
````

**表示結果:**

```bash
# 作業ディレクトリを確認して移動
TARGET_DIR=~/works/apple

if [ -d "$TARGET_DIR" ]; then
    echo "ディレクトリが見つかりました: $TARGET_DIR"
    cd "$TARGET_DIR"
else
    echo "エラー: $TARGET_DIR が存在しません"
    exit 1
fi
```

---

### よく使う言語指定

| 指定キーワード | 言語・用途 |
|----------------|------------|
| `bash` / `sh`  | シェルスクリプト・コマンド |
| `python`       | Python コード |
| `javascript`   | JavaScript |
| `json`         | JSON データ |
| `yaml`         | YAML 設定ファイル |
| `markdown`     | Markdown ソースを見せるとき |
| `diff`         | 差分表示 |
| （なし）       | 汎用・言語不明 |

## 5. リンク・画像

### リンク

**Markdown ソース:**

```markdown
[表示テキスト](URL)

[GitHub](https://github.com)
[企業概要を見る](./info/overview.md)
```

**表示結果:**

- `[GitHub](https://github.com)` → [GitHub](https://github.com)
- `[企業概要を見る](./info/overview.md)` → 同じリポジトリ内のファイルへのリンクとして機能

---

### ページ内リンク（アンカーリンク）

見出しに自動的にアンカーが生成されます。`#` の後に見出しテキストを小文字・ハイフン区切りで書きます。

**Markdown ソース:**

```markdown
[目次に戻る](#目次)
[コードブロックのセクションへ](#4-コードブロック)
```

---

### 画像

リンクと同じ構文に `!` を先頭に付けます。

**Markdown ソース:**

```markdown
![代替テキスト](画像のURL または相対パス)

![Apple ロゴ](./images/apple-logo.png)
```

## 6. テーブル

### 基本構文

列をパイプ `|` で区切り、2 行目にハイフン `-` で区切り線を書きます。

**Markdown ソース:**

```markdown
| 担当者 | 部署 | 連絡先 |
|--------|------|--------|
| 山田 太郎 | 法人営業 | yamada@apple.com |
| 鈴木 花子 | マーケティング | suzuki@apple.com |
```

**表示結果:**

| 担当者 | 部署 | 連絡先 |
|--------|------|--------|
| 山田 太郎 | 法人営業 | yamada@apple.com |
| 鈴木 花子 | マーケティング | suzuki@apple.com |

---

### 列の配置（左寄せ・中央・右寄せ）

**Markdown ソース:**

```markdown
| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| iPhone | 発売中   | 2024年 |
| iPad   | 発売中   | 2024年 |
```

**表示結果:**

| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| iPhone | 発売中   | 2024年 |
| iPad   | 発売中   | 2024年 |

## 7. 引用

### 基本的な引用（blockquote）

行頭に `> ` を付けると引用ブロックになります。

**Markdown ソース:**

```markdown
> これは引用ブロックです。
> 複数行にわたって書くことができます。
```

**表示結果:**

> これは引用ブロックです。
> 複数行にわたって書くことができます。

---

### blockquote の使いどころ

| 使いどころ | 例 |
|------------|----|
| 公式ドキュメントからの引用 | `> Apple は...` |
| 注意書き | `> **注意:** この操作は元に戻せません。` |
| ヒント・補足 | `> **コツ:** ブランチ名は作業内容を表す名前にする。` |

## 8. 実践：打ち合わせメモをブランチ経由で push する

ここまで学んだ Markdown 記法を使って、Apple との打ち合わせメモを作成します。

### ステップ 1: ブランチを作成する

```
> main を最新にして、feature/add-meeting-memo というブランチを作って
```

### ステップ 2: meetings フォルダと打ち合わせメモを作成する

```
> meetings/2025-01-meeting.md を作成して、以下の内容を書いて：

# Apple 打ち合わせメモ（2025年1月）

**日時**: 2025-01-15 14:00〜15:00
**場所**: Apple Japan 表参道オフィス
**参加者**: 山田 太郎（Apple 法人営業）、鈴木 花子（マーケティング）、我々チーム全員

---

## 議題

1. 2025年の取引方針の確認
2. 新製品（iPad Pro）の企業向け展開について
3. サポート体制の改善

---

## 決定事項

- [ ] iPad Pro の企業向け導入プランの見積もりを 1 月末までに提出
- [ ] サポート窓口を専用ダイヤルに変更（2月から）
- [ ] 次回打ち合わせは 2 月中旬に設定

---

## 共有事項

- Apple Japan の組織変更により、担当が山田さんに一本化された
- 新しい Volume Purchase Program（VPP）への移行を推奨された

---

## 次回までのアクション

| 担当 | 内容 | 期限 |
|---|---|---|
| 我々 | iPad Pro 見積もり作成 | 1/31 |
| 山田さん | VPP 移行手順書の送付 | 1/20 |
```

### ステップ 3: 変更をコミットする

```
> meetings/2025-01-meeting.md をコミットして。メッセージは「feat: 1月打ち合わせメモを追加」で
```

### ステップ 4: リモートに push して PR を作る

```
> push して PR を作って
```

### ステップ 5: GitHub で確認する

1. `https://github.com/[ユーザー名]/apple` を開く
2. `feature/add-meeting-memo` ブランチに切り替える
3. `meetings/2025-01-meeting.md` をクリックして、Markdown が正しくレンダリングされているか確認する

## 9. まとめ（記法早見表）

### Markdown 記法早見表

| 記法 | Markdown ソース | 表示結果・説明 |
|------|----------------|----------------|
| **見出し H1** | `# 見出し` | 最大の見出し |
| **見出し H2** | `## 見出し` | セクション見出し |
| **見出し H3** | `### 見出し` | サブセクション |
| **太字** | `**テキスト**` | **テキスト** |
| **斜体** | `*テキスト*` | *テキスト* |
| **太字＋斜体** | `***テキスト***` | ***テキスト*** |
| **取り消し線** | `~~テキスト~~` | ~~テキスト~~ |
| **インラインコード** | `` `code` `` | `code` |
| **箇条書き** | `- 項目` | 順序なしリスト |
| **番号付きリスト** | `1. 項目` | 順序ありリスト |
| **ネストリスト** | `  - 子項目`（2 スペース） | 入れ子リスト |
| **タスクリスト** | `- [ ] 未完了` / `- [x] 完了` | チェックボックス |
| **コードブロック** | ` ```bash ` ... ` ``` ` | シンタックスハイライト付きコード |
| **リンク** | `[テキスト](URL)` | クリック可能なリンク |
| **画像** | `![代替テキスト](URL)` | 画像の埋め込み |
| **テーブル** | `\| 列1 \| 列2 \|` | 表の作成 |
| **引用** | `> テキスト` | 引用ブロック |
| **水平線** | `---` | 区切り線 |
