## 1. Claude のリサーチ活用

### 単純検索との違い

Google 検索でできることと、Claude に依頼するリサーチの違いを理解することが第一歩です。

| 観点 | Google 検索 | Claude リサーチ |
|---|---|---|
| 操作 | キーワードを入力してリンクを開く | 自然言語で質問・依頼する |
| 情報の取得形式 | リンク一覧（自分で読む） | 要約・整理済みの文章 |
| 複数ソースの統合 | 手動で読み比べる必要がある | 自動的に統合・比較 |
| 追加質問 | 再検索が必要 | 同じ会話内で深掘り可能 |
| 出力のカスタマイズ | 不可 | Markdown・表・箇条書き等を指定可能 |
| 専門用語の解説 | 別途調べる必要がある | 調査と同時に解説してもらえる |
| 情報の鮮度 | リアルタイム | 学習データの締め切り日まで（※後述） |
| 金融・法律などの注意事項 | 自分で判断 | 免責事項を付記してもらえる |

### Claude リサーチが特に有効なシーン

- 「〇〇について概要を把握したい」という初期調査
- 複数の概念を比較・対比したいとき
- 調査結果を即座にレポートや Markdown にまとめたいとき
- 専門用語が多く、理解しながら調べたいとき

### Claude リサーチが苦手なシーン

- 今日・今週など「最新」の情報が必要なとき（Web 検索ツール併用で解決）
- 特定のウェブサイト内の情報を検索したいとき
- ローカルファイルや社内データベースを参照したいとき

## 2. Web 検索との連携

### MCP と WebSearch ツールとは

Claude は標準の言語モデルとしての知識だけでなく、**ツール（Tool）** を通じて外部情報にアクセスできます。

- **MCP（Model Context Protocol）**: Claude がファイルシステム、API、データベースなど外部リソースと連携するための標準プロトコル
- **WebSearch ツール**: MCP を通じて Claude がリアルタイムの Web 検索を実行できる機能

```
[あなた]
   ↓ 質問・依頼
[Claude]
   ↓ 必要と判断したらツールを呼び出す
[WebSearch / MCP Server]
   ↓ 検索結果・データを返す
[Claude]
   ↓ 結果を解釈・統合して回答
[あなた]
```

### Claude Code での Web 検索の使い方

Claude Code（VS Code 拡張または CLI）では、プロンプトに検索の意図を明示するだけで自動的にツールが使われます。

```bash
# Claude Code CLI で Web 検索を伴うリサーチを依頼する例
claude -p "今日のドル円レートと直近1週間のトレンドをまとめて" --output-format text
```

Claude.ai のウェブ版では、会話画面の入力欄左下に「ツール」アイコンがあり、**Web 検索を ON にする**ことでリアルタイム検索が有効になります。

### プロンプト設計のコツ

リサーチの精度は、プロンプトの書き方で大きく変わります。

#### 悪い例 vs 良い例

**悪い例 1: 漠然としすぎている**

```
Claude に聞く（悪い例）:
「株について教えてください」
```

問題点:
- 何を知りたいのかが不明
- 範囲が広すぎて表面的な回答になる
- 出力形式が指定されていない

**良い例 1: 目的・範囲・形式を明確にする**

```
Claude に聞く（良い例）:
「株式投資を始めたばかりの初心者向けに、
 以下の3点を箇条書きで説明してください。

 1. 株とは何か（1〜2文で）
 2. 株価が動く主な要因（3つ以内）
 3. 初心者が最初に確認すべき指標（PER・PBRを含む）

 専門用語には括弧で簡単な解説を付けてください。」
```

---

**悪い例 2: 検索キーワードをそのまま渡している**

```
Claude に聞く（悪い例）:
「Apple 決算 2024」
```

問題点:
- Google 検索に近い使い方で Claude の強みが活かせない
- 何をしてほしいかが不明

**良い例 2: 調査の目的とアウトプットを指示する**

```
Claude に聞く（良い例）:
「Apple（AAPL）の2024年度（FY2024）の決算について調べ、
 以下の形式でまとめてください。

 ## Apple FY2024 決算サマリー
 - 売上高: 〇〇億ドル（前年比 〇〇%）
 - 純利益: 〇〇億ドル（前年比 〇〇%）
 - 注目ポイント: 〇〇
 - 今後の見通し: 〇〇

 情報の出典と情報の鮮度（いつ時点のデータか）も明記してください。」
```

---

**悪い例 3: 一度の質問に詰め込みすぎる**

```
Claude に聞く（悪い例）:
「Appleと Googleと Microsoftの決算と株価と今後の見通しを全部教えてください」
```

問題点:
- 情報量が多すぎて回答が浅くなる
- 比較軸が不明確

**良い例 3: 段階的に深掘りする（会話を活用）**

```
ターン1:
「AppleとGoogleとMicrosoftを
 2024年度の売上高・純利益・営業利益率で比較する表を作ってください。」

↓ 表を受け取ったら

ターン2:
「この3社のうち、利益率の観点で最も優れているのはどれですか？
 その理由を3点挙げてください。」

↓ 分析を受け取ったら

ターン3:
「Microsoftについて、今後1年の事業見通しとリスク要因を
 箇条書きでまとめてください。」
```

### プロンプト設計チェックリスト

リサーチ依頼をする前に確認しましょう。

- [ ] 調べたいことが具体的に書かれているか
- [ ] 出力形式（表・箇条書き・Markdown など）を指定したか
- [ ] 必要な項目・観点をリストアップしたか
- [ ] 対象の期間・地域・対象者を絞り込んだか
- [ ] 情報の鮮度や出典の明記を求めたか

## 3. ファイナンシャル系リサーチ

### 企業情報・財務データの調べ方

Claude はビジネス・金融分野の幅広い知識を持っています。以下のプロンプトパターンを活用してください。

#### 企業概要を調べる

```
プロンプト例:
「トヨタ自動車（証券コード: 7203）について、
 以下の項目をまとめた企業プロフィールを作成してください。

 - 事業内容（50字以内）
 - 主要セグメント（売上比率順に3〜5個）
 - 主要ブランド・製品
 - 本社所在地・設立年
 - 従業員数（グループ全体）
 - 競合他社（国内・海外それぞれ2社ずつ）」
```

#### 財務指標を比較する

```
プロンプト例:
「以下の自動車メーカー3社について、
 最新の財務指標を比較表にまとめてください。

 対象: トヨタ自動車 / Honda / 日産自動車
 指標: PER / PBR / ROE / 配当利回り / 自己資本比率

 表の後に「投資初心者向けの各指標の見方」を
 1〜2文ずつ追記してください。」
```

#### 業績トレンドを把握する

```
プロンプト例:
「ソニーグループ（6758）の過去3年間（FY2022〜FY2024）の
 業績推移を以下の形式でまとめてください。

 | 年度 | 売上高 | 営業利益 | 純利益 | EPS |
 |------|--------|----------|--------|-----|
 | FY2022 | ... | ... | ... | ... |
 ...

 また、各年度のトピックス（重要な出来事・変化）を
 1〜2点ずつ箇条書きで補足してください。」
```

### ニュースとの組み合わせ

財務データだけでなく、最新ニュースと組み合わせることで、より実践的なリサーチができます。

```
プロンプト例（Web 検索ツール有効時）:
「以下の手順で調査してください。

 Step1: 直近1ヶ月以内のトヨタ自動車に関するニュースを
        3〜5件収集してください。

 Step2: 各ニュースが株価・業績に与える影響を
        プラス・マイナス・中立で判定し、理由を述べてください。

 Step3: 総合的な現状評価を100字以内でまとめてください。

 出典URL・日付も各ニュースに付けてください。」
```

### 注意点（情報の鮮度・正確性・免責事項）

> **重要な免責事項**
>
> Claude が提供する金融・投資に関する情報は、あくまで教育・参考目的のものです。
> 以下の点を必ず認識してください。
>
> - **情報の鮮度**: Claude の学習データには締め切り日（カットオフ日）があり、最新の株価・決算数値・金融規制は反映されていない場合があります。Web 検索ツールを使用した場合も、情報の正確性は保証されません。
> - **投資判断**: このドキュメントおよび Claude の回答は、投資アドバイスではありません。実際の投資判断は、信頼できる金融機関・公式データソース（金融庁・東証・各社 IR ページ等）を参照し、必要に応じてファイナンシャルプランナー等の専門家に相談してください。
> - **数値の確認**: Claude が提示した財務数値は、必ず一次ソース（有価証券報告書・決算短信等）で確認してください。

金融情報を扱うプロンプトには、必ず以下のような注意喚起の指示を含めましょう。

```
プロンプトに追加する一文の例:
「回答の末尾に、情報の鮮度と投資判断には一次ソースの確認が
 必要である旨の免責事項を追記してください。」
```

## 4. ニュース検索

### 最新ニュースの取得

Claude の Web 検索ツールを活用することで、最新ニュースの収集と整理を自動化できます。

#### 基本的なニュース収集プロンプト

```
プロンプト例:
「以下のテーマに関する直近7日間のニュースを収集し、
 重要度の高い順に5件まとめてください。

 テーマ: 日本の金融政策（日銀・金利）

 各ニュースの形式:
 ## [タイトル]
 - 日付: YYYY-MM-DD
 - 出典: メディア名（URL）
 - 概要: 2〜3文
 - 重要度: 高・中・低
 - 関連キーワード: #〇〇 #〇〇」
```

#### 複数ソースの比較

同じトピックでも、メディアによって論調や強調点が異なる場合があります。

```
プロンプト例:
「円安に関する最新ニュースを調べ、
 以下の観点でメディア間の論調の違いをまとめてください。

 比較するメディア: 日本経済新聞・NHK・Bloomberg Japan

 比較観点:
 - 円安の原因としてどの要因を強調しているか
 - 影響（プラス・マイナス）の取り上げ方の違い
 - 今後の見通しのトーン（楽観・悲観・中立）

 最後に「論調の違いから読み取れること」を
 3点箇条書きでまとめてください。」
```

#### 特定企業のニュースモニタリング

```
プロンプト例:
「[企業名] に関する直近2週間のニュースを
 以下のカテゴリに分類して整理してください。

 カテゴリ:
 - 決算・業績関連
 - 製品・サービス発表
 - 人事・組織変更
 - M&A・提携
 - 法規制・コンプライアンス
 - その他

 各カテゴリ内のニュースは日付の新しい順に並べ、
 出典と1行サマリーを付けてください。」
```

### 情報収集の自動化フロー

定期的なニュースモニタリングには、以下のようなルーティンが効果的です。

```
毎朝の情報収集ルーティン例:

1. Claude に依頼（所要時間: 約2分）
   └─ 「昨日の [テーマ] に関するニュースを3件まとめて」

2. 受け取った要約を調査メモに貼り付け
   └─ 後述の Markdown テンプレートを使用

3. 気になるニュースの出典 URL にアクセスして確認
   └─ Claude の要約はあくまで参考。一次ソースを確認する習慣を

4. 必要に応じて Claude に深掘りを依頼
   └─ 「このニュースが業界に与える影響を分析してください」
```

## 5. 調査結果を Markdown にまとめる

### Claude に Markdown 形式で出力させる

Claude は Markdown の書き方を熟知しており、指示するだけで適切な形式で出力してくれます。

#### 基本的な Markdown 出力指示

```
プロンプト例:
「以下の調査結果を Markdown 形式でまとめてください。

 ・見出しは ## と ### を使う
 ・重要な数値は **太字** で強調
 ・比較はテーブル形式
 ・注意点は > ブロック引用
 ・箇条書きは - を使う
 ・コードや固有名詞はバッククォート（`）で囲む

 調査内容: [ここに調べたいことを書く]」
```

#### 出力を指定する便利な指示フレーズ

| やりたいこと | 指示の例 |
|---|---|
| 表にまとめたい | 「〇〇を比較する Markdown テーブルを作ってください」 |
| 見出しで整理したい | 「H2・H3 の見出しで構造化してください」 |
| 重要箇所を強調したい | 「重要なキーワードを太字にしてください」 |
| 警告・注意を目立たせたい | 「注意点は `>` ブロック引用で記述してください」 |
| コードを含めたい | 「コマンド例はコードブロックで囲んでください」 |
| ファイルとして保存できる形式 | 「そのままファイルに保存できる Markdown を出力してください」 |

### 良い調査メモの構成例（テンプレート）

以下のテンプレートをコピーして使ってください。Claude に「このテンプレートに沿って調査結果をまとめて」と渡すこともできます。

```markdown
# [調査タイトル]

**調査日**: YYYY-MM-DD
**調査者**: [名前]
**目的**: [なぜこの調査を行ったか、1〜2文]

---

## エグゼクティブサマリー

[最重要な発見・結論を3〜5点の箇条書きで]

- 発見1
- 発見2
- 発見3

---

## 調査内容

### [サブトピック1]

[詳細な情報・データ]

| 項目 | 数値/内容 | 出典 |
|------|-----------|------|
| ... | ... | ... |

### [サブトピック2]

[詳細な情報・データ]

---

## ニュース・最新動向

| 日付 | タイトル | 出典 | 重要度 |
|------|---------|------|--------|
| YYYY-MM-DD | ... | [メディア名](URL) | 高/中/低 |

---

## 分析・考察

[データから読み取れること、重要な示唆]

> **注意**: [金融・法律情報の場合は免責事項をここに記載]

---

## 次のアクション

- [ ] [確認すべきこと1]
- [ ] [確認すべきこと2]
- [ ] [深掘りするべきトピック]

---

## 参考資料

- [資料名1](URL)
- [資料名2](URL)

---

*このメモは Claude によるリサーチを元に作成しました。*
*重要な判断を行う際は一次ソースを必ず確認してください。*
```

### テンプレートを Claude に渡す使い方

```
プロンプト例:
「以下のテンプレートに沿って、
 [調査テーマ] に関する調査メモを作成してください。

 ---テンプレート開始---
 # [調査タイトル]
 **調査日**: YYYY-MM-DD
 ...（テンプレートをそのまま貼り付ける）
 ---テンプレート終了---

 調査してほしいこと:
 - [具体的な調査ポイント1]
 - [具体的な調査ポイント2]」
```

### 調査メモを VS Code でプレビューする

作成した Markdown ファイルは、VS Code でリアルタイムにプレビューできます。

```bash
# ファイルを作成して VS Code で開く
code research_note.md
```

VS Code で開いたら:
1. 右上の「プレビュー」アイコンをクリック（または `Ctrl+Shift+V`）
2. Markdown がきれいにレンダリングされて表示される
3. 編集しながらリアルタイムでプレビューを確認できる

## 6. まとめ

### リサーチ → Markdown 化のフロー

以下は Claude を使ったリサーチから Markdown ファイルの保存まで、全体の流れを示したフロー図です。

```
┌─────────────────────────────────────────────────────────┐
│               Claude ディープリサーチ フロー              │
└─────────────────────────────────────────────────────────┘

  1. 調査テーマを決める
     └─ 「何を・なぜ・どんな形式で知りたいか」を明確にする

           │
           ▼

  2. プロンプトを設計する
     ├─ 目的・範囲・出力形式を指定
     ├─ 段階的に深掘りする設計
     └─ 免責事項・出典の明記を要求

           │
           ▼

  3. Claude に調査依頼
     ├─ [知識のみ] 学習データ範囲内の調査
     └─ [Web 検索有効] 最新情報も含む調査

           │
           ▼

  4. 回答を評価・深掘り
     ├─ 不明点を追加質問
     ├─ 数値・事実を一次ソースで確認
     └─ 必要なら別角度から再質問

           │
           ▼

  5. Markdown 形式で出力指示
     ├─ テンプレートを渡して構造を指定
     ├─ 見出し・表・引用ブロックを活用
     └─ 調査日・出典・免責事項を含める

           │
           ▼

  6. ファイルに保存
     ├─ VS Code で .md ファイルとして保存
     ├─ プレビューで見た目を確認
     └─ 必要に応じて手動で修正

           │
           ▼

  7. GitHub で共有・管理（→ 次のステップ）
     └─ git add / commit / push で公開
```

### リサーチ精度を上げる 5 つのポイント

1. **具体的に聞く**: 「〇〇について教えて」より「〇〇の△△を表形式でまとめて」
2. **段階的に深掘りする**: 1回の質問に詰め込まず、会話を重ねる
3. **出力形式を指定する**: Markdown・表・箇条書きなど形式を明示する
4. **一次ソースで確認する**: 特に金融・医療・法律は Claude の回答を鵜呑みにしない
5. **免責事項を付ける**: 金融情報には必ずリスク・注意事項を含める

### よくある失敗と対策

| 失敗パターン | 原因 | 対策 |
|---|---|---|
| 回答が浅い | 質問が漠然としている | 調査範囲・出力形式を明示する |
| 情報が古い | Web 検索なしで質問 | Web 検索ツールを有効にする |
| 数値が不正確 | Claude の学習データの限界 | 一次ソースで必ず確認する |
| 出力が長すぎる | 範囲を絞っていない | 項目数・文字数を指定する |
| フォーマットが崩れる | 形式を指定していない | Markdown テンプレートを渡す |

---

> **まとめ**
>
> Claude を使ったリサーチの本質は「質問の質」にあります。
> 目的・範囲・形式を明確にしたプロンプトを設計することで、
> 従来の検索では数時間かかっていた情報収集と整理を、
> わずか数分で完了できるようになります。
> ただし、特に金融・法律・医療の情報は必ず一次ソースで確認し、
> Claude の回答はあくまで「出発点」として活用してください。
